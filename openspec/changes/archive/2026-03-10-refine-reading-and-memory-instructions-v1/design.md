## Context

当前指令强调证据与记忆，但缺乏对“初析模板结构”和“QA 写入粒度”的明确约束，导致存储内容容易过短，进而影响最终学习笔记质量。

## Goals / Non-Goals

**Goals:**
- 固化半固定初析模板，提升初始理解输出完整度。
- 固化 QA 记忆写入策略，避免过度浓缩。
- 让笔记生成直接消费记忆字段，不再做二次压缩。

**Non-Goals:**
- 不迁移历史 schema 记录。
- 不新增数据库或外部存储。
- 不改变 memory 引擎三模式接口（instructions/update/read）。

## Decisions

1. QA 记录采用单条记录双字段方案。  
   备选：问题和回答拆成两条记录。  
   理由：保持每轮 QA 原子性，读取和关联更简单。

2. 长回答阈值固定为 1200，摘要目标区间 900-1200。  
   备选：更小摘要长度。  
   理由：靠近阈值以尽量保留信息密度。

3. 读取默认仅返回 schema v2。  
   备选：兼容并混读旧 schema。  
   理由：避免旧格式污染新流程，降低复杂度。

## Risks / Trade-offs

- [Risk] 900-1200 摘要仍可能偏长 → Mitigation：保持结构化要点组织，避免冗余叙述。
- [Risk] 忽略旧记录导致历史信息不可见 → Mitigation：明确策略并保留原文件，后续可单独做迁移 change。
