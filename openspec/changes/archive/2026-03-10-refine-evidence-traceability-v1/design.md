## Context

当前项目已具备证据验证和记忆写入能力，但证据原文在“回答输出 -> QA 记忆 -> 学习笔记”之间没有强一致约束，导致证据追溯链路不完整。

## Goals / Non-Goals

**Goals:**
- 在 S3 问答阶段强制每答附证据数组。
- 在 QA 记忆中强制保存结构化证据条目并保留 `source` 原文。
- 在学习笔记问答章节逐条回放证据原文。
- 通过 schema v3 明确新约束，默认忽略旧 schema 记录。

**Non-Goals:**
- 不扩展到 S1 初析或 S4 正文段落的证据强制输出。
- 不修改 `evidence_verifier.py` 的模式数量和总体判定策略。
- 不引入外部存储或历史数据迁移。

## Decisions

1. 证据强约束仅作用于 S3（Q&A）。
   理由：与现有验证脚本触发点一致，减少跨阶段行为分叉。

2. QA 记录升级到 schema v3，并新增必填 `evidence_items`。
   理由：通过版本边界避免旧记录缺字段影响新笔记生成质量。

3. 学习笔记问答章节按“每轮问答逐条列证据原文”输出。
   理由：保证可审计性与可复核性，避免二次改写导致证据失真。

## Risks / Trade-offs

- [Risk] 证据条目变长导致记忆文件增长更快。
  Mitigation：保留 `limit` 读取参数，按会话范围读取。
- [Risk] 旧记录默认被过滤后，历史会话在新规则下不可见。
  Mitigation：保留 `--include-legacy` 以支持回溯检查。
