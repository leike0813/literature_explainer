## Context

项目当前已经具备记忆记录与学习笔记生成的基础能力，但 note 阶段仍以原则性描述为主，缺少“生成是可选动作”“笔记以问答为核心”“论文部分只保留单段扩展摘要”等稳定规则，导致最终文档风格容易漂移。

## Goals / Non-Goals

**Goals:**
- 明确 note generation 是可选动作。
- 固定学习笔记的章节顺序和 Q&A 四项结构。
- 将论文原文总结收敛为单段扩展摘要。
- 让 note 阶段只消费结构化记忆字段，并在生成后写回 `stage=note` 记录。

**Non-Goals:**
- 不新增专门的 note 生成脚本。
- 不扩展非 note 阶段的新能力。
- 不把非 Q&A 部分细化成更多字段级模板。

## Decisions

1. 生成学习笔记是显式可选动作。  
   理由：用户可以只完成论文分析和问答，不必被迫进入笔记生成。

2. 学习笔记以问答为核心，论文部分只保留一段扩展摘要。  
   理由：当前 Skill 的核心价值在交互式答疑，而不是复写论文主体。

3. Note 阶段只依赖结构化记忆，并在生成后写回 `stage=note`。  
   理由：保持笔记生成链路可追踪，避免最后阶段重新依赖会话原文造成不稳定。

## Risks / Trade-offs

- [Risk] 论文部分过于精简可能丢失背景信息。  
  Mitigation：保留“扩展摘要”而不是压缩到一句话，并让 Q&A 成为主体承载详细内容。
- [Risk] 只消费结构化记忆会放大早期记忆写入质量问题。  
  Mitigation：沿用现有记忆规则，且 note `instructions` 明确要求直接使用结构化字段。
