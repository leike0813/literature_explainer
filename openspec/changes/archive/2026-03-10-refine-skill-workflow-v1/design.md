## Context

项目已具备最小 Skill 包骨架和初版 `SKILL.md`，但当前指令仍偏高层，存在执行口径波动。需要在不扩展目录和脚本的前提下，提升指令契约的可操作性与可验证性。

## Goals / Non-Goals

**Goals:**
- 将 `SKILL.md` 升级为可发布初版。
- 固化输入输出契约、证据分级规则、状态转换流程、学习笔记模板。
- 保持变更范围单一且可验证。

**Non-Goals:**
- 不新增 `references/`、`scripts/`、`assets/`、`agents/`。
- 不引入 Python 或其他辅助脚本。
- 不调整 OpenSpec schema 或归档旧 change。

## Decisions

1. 决策 1：仅通过 `SKILL.md` 提升能力，不引入脚本。  
   备选：新增脚本处理结构化输出。  
   理由：当前瓶颈在语义流程规范，而非确定性计算任务。

2. 决策 2：文案以中文为主，术语可保留英文。  
   备选：全英文或双语并行。  
   理由：与当前项目文档语言一致，降低维护成本。

3. 决策 3：将核心流程写成可检查条目。  
   备选：继续保留叙述式高层描述。  
   理由：可检查条目更利于一致执行和验收映射。

## Risks / Trade-offs

- [Risk] 规则增多导致文档变长 → Mitigation：保持条目化与最小必要信息，避免展开无关说明。
- [Risk] 无脚本支持时部分格式化工作仍依赖 Agent → Mitigation：先固化模板字段，后续再评估脚本化 change。
